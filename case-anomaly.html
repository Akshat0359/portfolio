01 — Context

Security at the Network Level

Network intrusion detection is a classic ML application domain where feature engineering and model selection have a direct impact on false positive rates. Security analysts are flooded with alerts — the goal is to surface the real threats, not drown in noise.

This project builds an anomaly detection pipeline using Zeek (a production-grade network analysis framework) as the traffic capture and feature extraction layer, combined with ML classifiers to flag abnormal connections.

02 — Problem

Why Network Anomaly Detection Is Hard

  • Network traffic is high-volume, unlabeled, and context-dependent — normal traffic patterns differ dramatically by time of day and workload
  • Feature engineering from raw PCAP/log data requires domain knowledge about protocols (TCP, UDP, DNS, HTTP)
  • Class imbalance is severe — anomalies are rare events in normal traffic distributions
  • High false positive rates make the tool unusable — analysts will disable it
03 — Architecture

Detection Pipeline

Network Traffic (PCAP / Live Interface) │ ▼ Zeek IDS → conn.log, dns.log, http.log, ssl.log │ ▼ Feature Extractor ├── Connection-level features (duration, bytes, packets) ├── Protocol-level features (DNS query types, HTTP methods) └── Statistical features (entropy, inter-arrival times) │ ▼ ML Classifier (Scikit-learn) ├── Isolation Forest (unsupervised anomaly detection) └── Random Forest (supervised, for labeled datasets) │ ▼ Streamlit Dashboard ├── Live alert feed ├── Anomaly score distribution └── Connection explorer
04 — Key Decisions

Technical Choices

  • Zeek over Wireshark/tcpdump: Zeek outputs structured logs (JSON/TSV) with protocol-level parsing already done — far more ML-ready than raw PCAP
  • Isolation Forest for unsupervised baseline: Works without labeled anomaly data, making it immediately applicable to new network environments
  • Streamlit for the UI: Enables rapid prototyping of interactive dashboards with minimal frontend overhead
  • Feature normalization: Min-max scaling applied per-feature to handle the extreme range of byte counts vs. packet counts

06 — Results

What Was Built

  • Full pipeline from Zeek log ingestion → feature extraction → ML classification → interactive alert dashboard

  • Feature engineering pipeline handling connection-level, protocol-level, and statistical features from Zeek logs
  • Streamlit dashboard with live anomaly feed, score distributions, and connection detail explorer
