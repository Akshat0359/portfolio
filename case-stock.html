01 — Context

Realtime Data Is a Different Problem

Most web development works with request-response cycles — user asks, server responds. Realtime stock dashboards are fundamentally different: the server pushes updates continuously, and the UI must reflect sub-second changes without re-rendering the entire component tree.

This project is a TypeScript-first broker client dashboard — demonstrating type-safe realtime data flows, WebSocket subscription management, and performant React rendering under continuous data updates.

02 — Problem

Realtime UI Challenges

  • WebSocket message handling is inherently untyped — without strict interfaces, data shape bugs are silent
  • Frequent state updates can cause React to re-render more than necessary, degrading performance
  • WebSocket connections need lifecycle management — reconnection, heartbeat, cleanup on unmount
  • Financial data display requires careful formatting — prices, percentages, deltas — and visual diff-highlighting
03 — Architecture

Data Flow Architecture

WebSocket Server (mock / real feed) │ (binary / JSON messages) ▼ WebSocket Client (TypeScript) ├── Type-safe message parsing (discriminated unions) ├── Reconnection logic with exponential backoff └── Custom useWebSocket hook │ ▼ React State Layer ├── Zustand store for ticker data ├── Selector-based subscriptions (minimal re-renders) └── Memoized price formatting utilities │ ▼ Dashboard UI Components ├── Live ticker strip (scrolling) ├── Portfolio summary cards ├── Price change indicators (delta highlighting) └── Chart components (line, candlestick)
04 — Key Decisions

Engineering Choices

  • TypeScript with discriminated unions for message types: Ensures every message variant is handled explicitly — prevents silent runtime failures on malformed data
  • Custom useWebSocket hook: Encapsulates connection lifecycle (open, message, close, reconnect) as a reusable React abstraction
  • Selector-based state subscriptions: Components subscribe only to the slice of state they use — prevents cascading re-renders on every price tick
  • CSS delta highlighting: Price changes trigger CSS class transitions (green/red flash) without DOM mutations — pure CSS animation performance
