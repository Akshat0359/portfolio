01 — Context

The Research Bottleneck

Technical research is fundamentally a multi-step process: formulate a question, retrieve relevant documents, synthesize across sources, evaluate credibility, and produce a structured output. Doing this manually across multiple tools is slow, context-lossy, and hard to reproduce.

The project aimed to build an agentic system that mirrors this process — not just a RAG pipeline, but a coordinated set of agents that plan, search, retrieve, and synthesize research end-to-end, with persistent memory across sessions.

02 — Problem

What Makes Multi-Agent Research Hard

  • Simple RAG pipelines have no planning layer — they cannot decompose a complex question into retrievable sub-questions
  • Stateless agents forget context between runs — you can't resume or build on prior sessions
  • Agent coordination in multi-step workflows often lacks fault tolerance — one agent failure breaks the entire chain
  • React-style UIs need real-time streaming updates from long-running async agents
03 — Architecture

System Design

The system follows a modular A2A (agent-to-agent) pattern with a central orchestrator and a shared MCP memory layer.

User Query │ ▼ Orchestrator Agent (LangGraph) ├── Planner Agent → Decomposes query into sub-tasks ├── Retrieval Agent → Fetches from tools/sources ├── Synthesis Agent → Merges + summarizes findings └── Critique Agent → Validates output quality ↕ All agents ↕ MCP Memory Layer (persistent state, session context) │ ▼ FastAPI Backend → WebSocket/SSE → React UI

The orchestrator uses LangGraph's stateful graph execution, allowing conditional branching, retries, and parallel agent execution where applicable. MCP memory acts as a shared knowledge store — agents read from and write to it throughout the session.

04 — Key Decisions

Engineering Choices That Mattered

  • LangGraph over bare LangChain: LangGraph's graph-based execution model gives explicit control over agent state transitions, making it easy to add conditional logic and human-in-the-loop steps
  • MCP for memory persistence: Model Context Protocol gives a structured contract for how agents read/write shared state — cleaner than ad-hoc in-memory stores
  • FastAPI for the backend: Async-native, type-checked with Pydantic, and natively supports both REST and SSE streaming — ideal for long-running agent jobs
  • React UI with streaming: Frontend subscribes to agent progress via server-sent events so users see intermediate results as they stream in, not just final output
  • Docker for environment isolation: Ensures consistent execution across local dev and any deployment target

06 — Results

What Was Built

  • A working multi-agent pipeline with four coordinated agents: planner, retrieval, synthesis, and critique
  • MCP memory layer enabling session persistence — agents can reference prior research context in new queries
  • FastAPI backend with streaming endpoint for real-time agent progress updates
  • React UI that streams intermediate and final agent outputs with clean UX
  • Modular directory structure (agents/, orchestrator/, memory/, tools/, ui/, api/) for clear separation of concerns
07 — Next Steps

What's Next

  • Improve critique agent with self-consistency scoring across multiple generation runs
  • Add a structured citation layer so every synthesized claim is traceable to a source
