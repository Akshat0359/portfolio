01 — Context

Financial Analysis Is Broken for Individuals

Analyzing a public company rigorously means reading SEC 10-K and 10-Q filings, tracking market data, monitoring news sentiment, and synthesizing across all of this into a coherent investment or risk view. This takes analysts hours. Retail investors and engineers don't have hours.

FinSight AI was built to automate this pipeline end-to-end — from raw SEC data ingestion to a final structured financial intelligence report — using a LangGraph agent system with Gemini AI at its core.

02 — Problem

The Technical Challenges

  • SEC EDGAR filings are long-form, semi-structured documents — naive embedding + retrieval fails on them without careful chunking and metadata tagging
  • Market data and news arrive in fundamentally different schemas than SEC documents — multi-source fusion requires a routing layer
  • LLMs hallucinate financial figures without grounding — RAG is necessary but not sufficient; the retrieval must be high-precision
  • Report delivery needs to be consumable by non-engineers — a raw API response is not enough
03 — Architecture

Data Flow & Agent Design

SEC EDGAR API → Raw Filing Parser → Chunker → Vector DB Market Data API → Normalizer ──┐ News RSS/API → Sentiment Parser ──┤ ▼ LangGraph Agent System ├── Data Fusion Agent ├── RAG Query Agent (Gemini AI + Vector DB) ├── Analysis Agent (ratios, trends, flags) └── Report Agent (structured output) │ MCP Server (context persistence) │ Streamlit Dashboard

The MCP server handles cross-session context — so if you analyzed Apple last week, the system can reference that analysis when comparing it to Microsoft today. The Streamlit dashboard presents the final report with charts, tables, and narrative sections.

04 — Key Decisions

What Drove the Design

  • FastMCP for the MCP server: FastMCP provides a clean Python interface for MCP servers with automatic type coercion, eliminating boilerplate
  • LangGraph checkpointing: Used LangGraph's SQLite-backed checkpointing to persist agent state between runs — essential for multi-session workflows
  • Gemini AI as the core LLM: Gemini's long context window is particularly valuable for SEC filings which can exceed 100 pages
  • Streamlit for delivery: Chosen explicitly for rapid iteration — gets a working interactive dashboard without a full React frontend
  • RAG with metadata filtering: Each chunk carries metadata (company ticker, fiscal year, section) enabling precise retrieval without scanning the entire corpus

06 — Results

What Was Delivered

  • End-to-end pipeline from raw SEC EDGAR filing → structured financial intelligence report
  • Four coordinated LangGraph agents: data fusion, RAG query, analysis, and report generation
  • MCP server for cross-session context memory, enabling comparative analysis across companies and time periods
  • Streamlit dashboard with charts, financial ratio summaries, and narrative report sections
  • Docker Compose setup for reproducible local and cloud deployment
